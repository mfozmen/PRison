## Known limitation

- **"Expected" required checks aren't visible via the API for non-admins.** A
  repo's required-but-not-yet-reported checks (e.g. a manual qa/smoke or
  automation step) live in branch-protection settings GitHub only exposes to
  repo admins. As an org *member* we can't fetch their names — verified across
  GraphQL `statusCheckRollup`/`branchProtectionRule`, REST
  combined-status/check-runs/branch-protection (404), and rulesets. The web UI
  shows them via a privileged path the public API doesn't replicate. Mitigated
  by: (a) `mergeStateStatus` reliably flags the PR as blocked with a note + a
  link to the GitHub Checks tab, and (b) **Tracked checks** (#40) — you name the
  expected checks and they show as "Awaiting" until they report.

## Parked (revisit later)

- **Surface partial-data drops.** `ghQuery` silently drops the parts that error
  (e.g. one org transiently fails) and returns partial data with no indication,
  so a refresh can quietly show incomplete lists. Plan: signal when a result was
  partial and show a "⚠ Some organizations couldn't be loaded — Retry" notice
  (optionally one auto-retry). Does not fix GitHub's search-index eventual
  consistency (a later refresh handles that).

## Notes

- "Remember the last selected org" is already implemented (`localStorage`
  `prison.org`); it was masked by the All-load bug, now fixed.
- Orgs that forbid classic PATs (e.g. an SSO-restricted work org) are reached via
  the GitHub CLI token (org-approved) or a fine-grained PAT — covered by the CLI
  sign-in.
