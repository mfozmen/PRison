# PRison v2 — Feature Roadmap

Tracks the post-MVP features requested while making PRison usable against a real
(SSO-restricted) work setup. Each lands as its own PR.

## Done

| PR | Feature |
| --- | --- |
| #21 | **PAT auth** — token in an AES-256-GCM-encrypted httpOnly cookie; OAuth/NextAuth removed (org-restricted orgs can't approve a third-party OAuth app) |
| #22 | **Docker support** — multi-stage, Next.js standalone, non-root |
| #23 | **Partial-data tolerance** — the "All" view no longer fails entirely when one org forbids the token; it shows the orgs you can access |
| #24 | **One-click GitHub CLI sign-in** — server runs `gh auth token` (loopback-only), no manual paste; PAT now optional. README reordered: CLI first, PAT fallback |
| #26 | **Draft status + clearer details** — a "Draft" badge + "Hide drafts" toggle; the stuck list shows which checks are failing/pending as colored chips (names), not just counts |
| #27 | **Layout overhaul + grouping** — responsive two-column layout (lists side-by-side on desktop, stacked on mobile), wider container, and a Flat / By-repo grouping toggle |

## Considered / open

- **"All my open PRs" view** — personal PRs are correctly excluded from "stuck on
  checks" because they aren't stuck (e.g. green or no-checks). A broader
  "all open PRs" view was offered; awaiting a decision.

## Notes

- "Remember the last selected org" is already implemented (`localStorage`
  `prison.org`); it was masked by the All-load bug, now fixed.
- Orgs that forbid classic PATs (e.g. an SSO-restricted work org) are reached via
  the GitHub CLI token (org-approved) or a fine-grained PAT — covered by the CLI
  sign-in.
